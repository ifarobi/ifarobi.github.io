Hi Hugo

June 13, 2016

Awal kisah, saya pengen mulai belajar menulis, pengen bagi-bagi sesuatu apa yang saya punya di pikiran maupun di hati #tsah.

Okesip, mulai deh mikir pake platform blog apa.

Dulu pernah pakai Ghost, walaupun cuma bertahan beberapa minggu karena kurang aktif menulis. Platform blog ini sangat bagus bagi penggemar NodeJS. Sudah lengkap dengan dashboard admin dan editornya yang menggunakan Markdown.

Sayangnya Ghost memakan storage cukup banyak terutama di bagian pakage-nya (yah begitulah npm #hiks). Dan akhirnya saya memutuskan untuk memakai static web generator, itung-itung saya gak perlu mikirin keamanannya soalnya statis hihi…

Disini awal mula saya berjodoh dengan Hugo. Doakan semoga langgeng dan bisa istiqomah ya #eh.

Hugo adalah project open source dibuat menggunakan bahasa pemrograman Go. Cara kerja Hugo yaitu dengan mengkonversi konten-konten yang telah kita tulis dengan menggunakan Markdown dan juga template web yang telah kita pilih menjadi halaman HTML statis. Semua konten dari blog tersimpan di folder content. Berikut contoh jika kita ingin membuat konten yang berjudul Ini Post Pertama.

$ hugo new site blog
$ hugo new post/ini-post-pertama.md
$ hugo server --buildDrafts

0 of 1 draft rendered
0 future content
0 pages created
0 paginator pages created
0 tags created
0 categories created
in 9 ms
Watching for changes in /home/ifarobi/Hugo/bookshelf/{data,content,layouts,static}
Serving pages from memory
Web Server is available at http://localhost:1313/ (bind address 127.0.0.1)
Press Ctrl+C to stop

Dari hasil diatas bisa kita lihat kalau ada draft yang tidak ikut di-render. Maka kita bisa menggunakan --buildDrafts parameter untuk me-rendernya.

$ hugo server --buildDrafts

Contoh diatas hanyalah sebatas membuat post baru dan menampilkannya. Sebelumnya kita harus menggunakan template. Untuk template bisa kita pilih disini. Setelah template ter-install kita bisa langsung menggunakan parameter --theme atau langsung mengisi di konfigurasi config.toml.

$ hugo server --theme=hugo_theme_robust --buildDrafts

Konten yang kita buat sebelumnya tersimpan dengan status draft. Untuk mengubah status menjadi publish :

$ hugo undraft content/post/ini-post-pertama.md

Jika semua sudah siap untuk di-generate dan di publish.

$ hugo

Command diatas akan meng-generate template yang telah di setting di config.toml dan konten-konten yang telah dibuat. Setelah HTML ter-generate, kita bisa menempatkannya ke Github, Bitbucket, maupun shared hosting, karena ini hanya HTML biasa. Mantap to?

Hugo juga menyediakan template yang cukup banyak, dan sangat mudah untuk di customize. Setiap template mengambil parameter konfigurasi yang ada dalam file config.toml.

Ohiya ini bukan tulisan sebuah tutorial ya, ini cuma memperkenal kan Hugo saja. Untuk tutorial lebih detilnya langsung saja dilihat di dokumentasinya.
